Learning With an Answer Key

Supervised learning starts with known examples, tunes a rule from them, then uses that tuned rule on new examples.

The practical question is: what answer are we trying to learn? A category, a number, a nearby neighbor, a margin, or a layered pattern?

01

Classification

Pick a category, like spam or not spam, based on labeled examples.

02

Linear Regression

Fit a line that predicts a number, like price, score, or demand.

03

k-Nearest Neighbors

Look at nearby examples and vote from the local neighborhood.

04

SVM

Find a separating boundary with as much margin as possible.

05

MLP / DNN

Stack simple transformations to learn curved decision surfaces.

Interactive sketch

A tiny classifier learns a boundary.

Each training example on the graph has two properties: a horizontal position and a vertical position. All of the examples are labeled “coral” or “cyan” based on which cluster they are near.

This classifier attempts to draw a line that lets it correctly guess the color of each dot based on which side of the line the dot sits. Step through training and watch the decision line move as the algorithm adjusts after incorrect guesses. Add additional points of either color yourself to bend the lesson.

One step

  1. Read one labeled point.
  2. Guess which side of the line it belongs on.
  3. Compare the guess with the known label.
  4. If wrong, nudge the line toward the right answer.

This sketch uses a simple linear classifier inspired by the perceptron. “Linear” means it tries to separate the two groups cleanly with a single straight line. The line is the current decision boundary: points on one side are guessed as cyan, and points on the other side are guessed as coral. If the machine guesses the wrong color, the angle of the line changes to include that new information. This happens by updating weights on the features of the training examples. These weights are internal to the machine.

A feature is a piece of information that describes the examples the machine can read. Every example has the same set of features, but each example may have different values. Here each point has two features: its horizontal position and its vertical position, plus a label for its color. In a real dataset, features might be a house's square footage, an email's word count, or a customer's number of visits.

The equation w ← w + η(y - ŷ)x says: if the guess is wrong, adjust the weights to redraw the boundary line. w is the current set of knobs, η (eta) is the learning rate, or how big of an adjustment to make after each wrong guess, y is the label on the example, ŷ (y-hat) is the machine's guess, and x is a list of the point's features, called a feature vector.

In simple terms: “guessed wrong? move the line a little toward dividing cyan from coral based on the examples seen so far.” We repeat this process over and over, guessing and adjusting until the machine performs well enough. One full pass through all of the training examples is called a training epoch. With large datasets, it may take many epochs to get a useful model.

Try placing a cyan dot or three in the cluster of coral dots, or coral dots inside the cyan cluster, and watch how the mistakes and accuracy respond.
