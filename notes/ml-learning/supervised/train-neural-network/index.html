Training a Neural Network

Training is where the machine changes the numbers inside the model, using answer-key examples to make future predictions less wrong.

In the Build lesson, the network could pass signals forward and make a prediction. The inputs were fixed facts about a title and audience. The output was a guessed completion rate. Between those two ends were the model’s adjustable numbers: the weights on each input and the bias that shifts the neuron’s baseline.

Those adjustable numbers are called parameters. A weight is a parameter. A bias is also a parameter. “Parameter” is the umbrella word for any internal number the model is allowed to learn from data. Training is the process where the machine changes those parameters after it sees examples with known answers. That is the learning in machine learning.

For one example, the loop is small: make a prediction ŷ, compare it with the known answer y, turn that miss into one loss number, calculate how each parameter affects that loss, then let the update rule nudge the parameters so the same example would be predicted a little better next time.

Loss is the thing training tries to minimize. The residual tells us the direction and size of a miss. The loss function turns that miss into a score the optimizer can chase downward. In this lesson, the score is squared error for one example and Mean Squared Error across many examples. Lower loss means the model's predictions are closer to the answer key under that rule.

The object we are training

The machine learns by rewriting parameters.

The forward pass uses the current parameters to make ŷ (“y-hat”). Loss compares that prediction with y, the answer key, and gives the optimizer one number to minimize. Backpropagation computes a gradient for each parameter: a signed number that says how loss would change if that parameter moved. Gradient descent uses those gradients to update the model parameters themselves. It is not updating the gradient, and it is not comparing against the original baseline model forever. We are not choosing the new weights by hand; the training algorithm is.

Bias

Bias, without the fog

A bias is a learned offset. In this tiny model there is one output unit, so there is one bias for that unit. In a larger neural network, each neuron usually has its own bias parameter. The bias is not chosen by a data scientist one neuron at a time; the scientist chooses the architecture and training setup, then gradient descent adjusts each bias along with the weights. “Adjusting the baseline” means shifting what the neuron would output before the input features push it up or down.

Residuals

Every example gives the model a correction signal.

A positive residual means the model predicted too low. A negative residual means it predicted too high. The gradient combines that miss with the example’s feature values.

What to watch

  • Tiny learning rates move steadily but slowly; useful rates drop loss faster.
  • Oversized rates can bounce around the valley or make held-out loss worse.
  • Residual bars shrink when the model is learning the training examples.
  • Held-out loss is the early warning that “better on practice cards” is not always better on new cards.

How it is trained

Predict. Measure. Update.

The lab uses Mean Squared Error and gradient descent. The formulas below are the same operations used by the demo.

01 · Residual
residual = y − ŷ

y is the known answer. ŷ (“y-hat”) is the prediction. The residual says which way the prediction missed.

02 · Mean Squared Error
MSE = (1/n) Σ(yi − ŷi)²

Σ (“capital sigma”) means add the squared misses. n is the number of examples. Training tries to make this average smaller.

03 · Gradient descent
w ← w − η(∂L/∂w)

η (“eta”) is the learning rate. ∂L/∂w (“partial L over partial w”) is how sensitive loss is to this weight.

Generalization

The goal is not to memorize the practice set.

Training loss measures performance on examples used to update the model's parameters. Held-out loss measures examples kept aside: the model predicts them, but their errors do not change the weights or bias.

“Learning generalizes” means the model is not merely getting better at the exact practice cards it has already seen. It has found a pattern that also helps on examples that did not participate in the updates. When training loss keeps falling but held-out loss rises, the model may be overfitting: memorizing quirks of the practice set instead of learning a reusable relationship.

That is why modern training workflows separate training examples, validation or held-out examples, and final test examples. The model should earn trust on data it did not use to tune itself.

About “reward” language

This lesson uses loss, not reward.

In supervised learning, the usual signal is a loss function: a number the optimizer tries to minimize. “Reward” belongs most naturally to reinforcement learning, where an agent takes actions and receives scores to maximize. Modern chat systems may also use human preferences or reward models during alignment steps such as RLHF, but that is a later training stage, not the basic supervised loop shown here.

Words like learn, reward, and reason are useful shorthand, but they can sound more human than the mechanism is. On this page, “learning” means automatic parameter adjustment that reduces measured loss.

Key terms

The vocabulary of learning from error.

These terms show up across neural networks, linear models, and many other supervised methods.

Residual
The difference between the known answer and the prediction.
Loss function
A rule that turns prediction error into a number the model can minimize.
Gradient
The slope of loss with respect to a parameter.
Gradient descent
An update rule that steps parameters opposite the gradient.
Learning rate
The hyperparameter controlling the size of each update step.
Parameter
An internal number the model can learn, such as a weight or bias.
Bias
A learned offset for a neuron, adjusted by training along with the weights.
Backpropagation
The efficient backward pass that calculates gradients for the parameters.
Epoch
One training pass through every example in the training set.
Held-out examples
Examples not used to update parameters, used to check whether the learned pattern transfers.
Reward
A score to maximize in reinforcement learning; different from supervised loss minimization.
